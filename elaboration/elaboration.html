The Problem with "Type in Type" and a resolution thereof

Abstract

In "A Tutorial Implementation of a Dependently Typed Lambda Calculus" (A. Löh et al., 2010) a simple dependently typed lambda calculus and a straightforward implementation of type inference and checking for it are presented. In keeping the type rules and implementation simple, they made the deliberate choice to include the rule "type in type", which is commonly known to be inconsistent. Here, this inconsistency will be explored and an extension to the type rules and it's implementation will be presented which resolve it.

Overview

The type system for which A. Löh et al. [Löh 2010] presented their straightforward implementation in Haskell, which we shall call λΠττ\lambda_{\Pi}^{\tau \tau}λΠττ​ and which will be taken as the basis of the following discussion, is a direct extension of the simply typed lambda calculus (STLC), with function types, τ→τ′\tau \rightarrow \tau'τ→τ′, extended to dependent function types, ∀x:ρ.ρ′\forall x : \rho . \rho'∀x:ρ.ρ′, and the distinction between ordinary terms and type terms being dissolved. From this results the necessity for a term which is the type of all types, ∗\ast∗, which in turn of course also requires a type itself. The perhaps most straightforward choice to be made here is to consider the type of ∗\ast∗ to be ∗\ast∗ itself ("type in type"). Just like in Martin-Löf's 1971 "A Theory of Types" [Martin-Löf 1971] this is in fact the choice that A. Löh et al. made. However, contrary to Martin-Löf in 1971, they did so in full knowledge that this results in an inconsistent type system, as was shown by Girard in 1972 [Girard 1972], to keep the type system and it's implementation simple.

The inconsistency arising from ∗:∗\ast : \ast∗:∗ will be the focus of the first part of this paper. However, instead of Girard's original proof, a much simplified construction due to Hurkens [Hurkens 1995] will be discussed, which in the following shall be called "Hurkens' paradox". From this paradox we will arrive at (relatively) compact concrete term of type ∀A:∗.A\forall A: \ast. A∀A:∗.A, which should be impossible in a consistent type system, given that by applying this term to any possible type we receive a term of that type, including any definitionally empty types. So, looked at through the Curry-Howard Correspondence, where we take types to represent propositions and terms to represent proofs, this entails that we can provide a proof for every possible proposition, which clearly would make such an implementation of little use as the basis for a proof assistant.

While there are different ways of resolving the inconsistency arrising from ∗:∗\ast : \ast∗:∗, in the second part of this paper, one possible solution, namely a "hierarchy of sorts" [Coquand 1986] will be introduced, and an extended version of the lambda calculus and typing implementation presented by A. Löh et al., which we shall call λΠω\lambda_{\Pi}^{\omega}λΠω​, will be presented.

Hurkens' Paradox

In 1995, Antonius J.C. Hurkens derived, based upon work by Girard [Girard 1972] and Coquand [Coquand 1991], a relatively compact term of type ⊥\bot⊥ in λU−\lambda U^-λU− [Hurkens 1995]. While the type system of λU−\lambda U^-λU− goes beyond the type system of λΠττ\lambda_{\Pi}^{\tau \tau}λΠττ​, his construction can be followed one-to-one, giving us a term of type ⊥\bot⊥ in λΠττ\lambda_{\Pi}^{\tau \tau}λΠττ​, proving the type system's inconsistency, which we shall do in the following.

Though Hurkens showed two different approaches to simplifying Girard's paradox, the one for which he provided a complete term of type ⊥\bot⊥ is based upon the concept of "powerful universes", and will be the one explored here.

While the goal in the end will be to implement the paradox in the mentioned implementation of λΠττ\lambda_{\Pi}^{\tau \tau}λΠττ​, for readability and convenience, in the following, the syntax of the dependently typed programming language Agda [?] will be used in the explanation of the paradox.

For a complete and annotated Agda source file implementing Hurkens' paradox, a translation thereof into the abstract syntax of our implementation of λΠττ\lambda_{\Pi}^{\tau \tau}λΠττ​ in Haskell, and also the fully expanded term of type ⊥\bot⊥ in Haskell, refer to the code repository associated with this paper [?].

In the following, each type theoretic proof, given in Agda syntax, will be accompanied with a corresponding elaborated set theoretic proof which shall aid in understanding of the proof term and it's derivation.

Basic Definitions

In absence of record types, we will, as is common, define the empty type ⊥\bot⊥ and negation ¬\neg¬ as follows:

⊥ : Set
⊥ = (A : Set) → A

¬ : Set → Set
¬ P = P → ⊥

A term of type ⊥\bot⊥ would have to be a function that could produce a term for any possible type, i.e. a proof of every possible proposition. Therefore this type has to be empty for the type system to be consistent.

A term of type ¬P\neg P¬P for some proposition PPP is simply a function which, if a proof of PPP were given, would produce a term of type ⊥\bot⊥. Therefore, if we can construct a term of type ¬P\neg P¬P, then this means we can not possibly produce a term of type PPP, meaning that the proposition P must not be true. At least that is the case in a consistent type system.

This gives us the first hint as to how we could derive a term of type ⊥\bot⊥. If we can come up with some proposition PPP for which we can both derive a term of type PPP and of type ¬P\neg P¬P, then we simply have to apply ¬P\neg P¬P to PPP and we will have our term of type ⊥\bot⊥ in hand. In fact, this will be exactly what we shall do in the end, however, first we have to come up with such a proposition.

For ease of readability and conceptual understanding, we shall also define a function for the type of all propositions over some type AAA. From a set-theoretic perspective, this is to be understood as the set of all subsets for some set AAA, i.e. it's powerset:

℘ : Set → Set
℘ A = A → Set

This powerset function will be made extensive use of in the following to allow us to build up our paradox.

A Powerful Universe

The first significant definition for Hurkens' Paradox is an instance of a powerful universe, which we shall consider essentially plucked out of thin air, in the knowledge that it will allow us to derive our contradiction:

U : Set
U = (A : Set) → (℘ (℘ A) → A) → ℘ (℘ A)

τ : ℘ (℘ U) → U
τ ppU A ppA→A pA = ppU (λ u → pA (ppA→A (u A ppA→A)))

σ : U → ℘ (℘ U)
σ u = u U τ

This triple of (U,σ,τ)(U,\sigma,\tau)(U,σ,τ) we consider to be powerful, since it satsifies, in set theoretic terms, the following property:

∀C∈℘(℘U):σ(τC)={X∣{y∣τ(σy)∈X}∈C}\forall C \in \wp (\wp U): \sigma (\tau C) = \left \{ X | \left \{ y | \tau (\sigma y) \in X \right \} \in C \right \} ∀C∈℘(℘U):σ(τC)={X∣{y∣τ(σy)∈X}∈C}

We will not concern ourselves with translating this property into type theory or proving that this property holds for our (U,σ,τ)(U, \sigma, \tau)(U,σ,τ) (see Hurkens' original derivation [Hurkens 1995] for some elaboration on the definition of a powerful universe), since such a proof term will not be necessary in constructing our paradox. Rather we will implicitly use this property as it arises from the behaviour of τ\tauτ and σ\sigmaσ as defined above.

Inductive Subsets and Well Founded Elements

For subsets of UUU we define the following proposition:

inductive' : ℘ (℘ U)
inductive' pU = ((u : U) → σ u pU → pU u)

In set theoretic terms this means that for some subset pUpUpU of UUU, we consider pUpUpU to be inductive iff the following property holds:

∀u∈U:(pU∈σu⇒u∈pU)\forall u \in U: \left ( pU \in \sigma u \Rightarrow u \in pU \right ) ∀u∈U:(pU∈σu⇒u∈pU)

Using this property over subsets of UUU, we define a proposition over elements of UUU:

well-founded : ℘ U
well-founded u = (pU : ℘ U) → inductive' pU → pU u

In set theoretic terms this means that we consider some element uuu of UUU to be well founded iff it is in every inductive subset of UUU.

A Paradoxical Element

With τ\tauτ from our definition of UUU as a powerful universe, we pick out a specific element in UUU for which we can show that it simultaneously is well founded, and isn't well founded, which will give us the contradiction we seek:

Ω : U
Ω = τ inductive'

This means that in set theoretic terms, we consider Ω\OmegaΩ to be τ{pU∈℘U∣pU is inductive}\tau \left \{ pU \in \wp U | pU \, is \, inductive \right \}τ{pU∈℘U∣pUisinductive}.

The Paradoxical Element is well founded

The proof that Ω\OmegaΩ is well founded is relatively straightforward:

well-founded-Ω : well-founded Ω
well-founded-Ω pU ind-pU = ind-pU Ω (λ u → ind-pU (τ (σ u)))

In set theoretic terms, we need to show that for any inductive subset pUpUpU of UUU, Ω\OmegaΩ is in pUpUpU. Knowing that pUpUpU is inductive, this means that we need to show that pUpUpU is in σΩ\sigma \OmegaσΩ.

Since UUU is powerful, σΩ={pU∈℘U∣{u∈U∣τ(σu)∈pU} is inductive}\sigma \Omega = \left \{ pU \in \wp U | \left \{ u \in U | \tau (\sigma u) \in pU \right \} \, is \, inductive \right \}σΩ={pU∈℘U∣{u∈U∣τ(σu)∈pU}isinductive}. So to show that our pUpUpU is in σΩ\sigma \OmegaσΩ, we need to show that {u∈U∣τ(σu)∈pU}\left \{ u \in U | \tau (\sigma u) \in pU \right \}{u∈U∣τ(σu)∈pU} is inductive.

Since pUpUpU is inductive, we known that for any u∈Uu \in Uu∈U, if pUpUpU is in σ(τ(σu))\sigma (\tau (\sigma u))σ(τ(σu)), then τ(σu)\tau (\sigma u)τ(σu) is in pUpUpU. But this already is exactly what we need to show to prove that {u∈U∣τ(σu)∈pU}\left \{ u \in U | \tau (\sigma u) \in pU \right \}{u∈U∣τ(σu)∈pU} is inductive, therefore our proof is complete.

But also not well founded

To construct the proof that Ω\OmegaΩ is at the same time not well founded, we define one more concept:

_<_ : U → U → Set
v < u = (pU : ℘ U) → σ u pU → pU v

For some u∈Uu \in Uu∈U, we consider some v∈Uv \in Uv∈U to be a predecessor of uuu iff for every subset pUpUpU of UUU, pUpUpU being in σu\sigma uσu implies that vvv is in pUpUpU.

With this concept, we define ourselves one specific subset of UUU, which will turn out to be inductive:

Δ : ℘ U
Δ u = ¬ (τ (σ u) < u)

So in set theoretic terms, Δ={u∈U∣τ(σu)≮u}\Delta = \left \{ u \in U | \tau (\sigma u) \nless u \right \}Δ={u∈U∣τ(σu)≮u}.

We prove that Δ\DeltaΔ is inductive:

inductive-Δ : inductive' Δ
inductive-Δ u σuΔ τσu<u = τσu<u Δ σuΔ (λ pU → τσu<u λ w → pU (τ (σ w)))

To show that Δ\DeltaΔ is inductive, we need to show that for any u∈Uu \in Uu∈U, if Δ\DeltaΔ is in σu\sigma uσu then uuu is in Δ\DeltaΔ, so τ(σu)≮u\tau (\sigma u) \nless uτ(σu)≮u.

So for any u∈Uu \in Uu∈U with Δ∈σu\Delta \in \sigma uΔ∈σu, we assume τ(σu)<u\tau (\sigma u) < uτ(σu)<u and arrive at a contradiction from this assumption as follows:

τ(σu)<u\tau (\sigma u) < uτ(σu)<u means that for any pUpUpU in σu\sigma uσu, τ(σu)\tau (\sigma u)τ(σu) is in pUpUpU. If we take for pUpUpU Δ\DeltaΔ itself, this means that τ(σu)\tau (\sigma u)τ(σu) is in Δ\DeltaΔ, so τ(σ(τ(σu)))≮τ(σu)\tau (\sigma (\tau (\sigma u))) \nless \tau (\sigma u)τ(σ(τ(σu)))≮τ(σu).

On the other hand we can show that τ(σ(τ(σu)))<τ(σu)\tau (\sigma (\tau (\sigma u))) < \tau (\sigma u)τ(σ(τ(σu)))<τ(σu) as follows:

For any subset pUpUpU of UUU we have to show that if pUpUpU is in σ(τ(σu))\sigma (\tau (\sigma u))σ(τ(σu)), then τ(σ(τ(σu)))\tau (\sigma (\tau (\sigma u)))τ(σ(τ(σu))) is in pUpUpU. However, since UUU is powerful, this simplifies to having to show that if {w∣τ(σw)∈pU}\left \{w | \tau (\sigma w) \in pU \right \}{w∣τ(σw)∈pU} is in σu\sigma uσu, then τ(σu)\tau (\sigma u)τ(σu) is in {w∣τ(σw)∈pU}\left \{w | \tau (\sigma w) \in pU \right \}{w∣τ(σw)∈pU}. But that follows directly from our initial assumption that τ(σu)<u\tau (\sigma u) < uτ(σu)<u. Therefore our proof is complete.

With Δ\DeltaΔ and the knowledge that it is inductive in hand, we can at last prove that Ω\OmegaΩ is not well founded, which will complete the contradiction we seek:

¬well-founded-Ω : ¬ (well-founded Ω)
¬well-founded-Ω wfΩ = wfΩ Δ inductive-Δ (λ pU → wfΩ (λ w → pU (τ (σ w))))

To show that Ω\OmegaΩ is not well founded, we assume that it is, and will from this derive a contradiction:

Since Δ\DeltaΔ is inductive and Ω\OmegaΩ well founded, this means that Ω\OmegaΩ is in Δ\DeltaΔ, and therefore τ(σΩ)≮Ω\tau (\sigma \Omega) \nless \Omegaτ(σΩ)≮Ω. On the other hand, we can show that τ(σΩ)<Ω\tau (\sigma \Omega) < \Omegaτ(σΩ)<Ω as follows:

For any subset pUpUpU of UUU we have to show that if pUpUpU is in σΩ\sigma \OmegaσΩ, then τ(σΩ)\tau (\sigma \Omega)τ(σΩ) is in pUpUpU. However, since UUU is powerful and Ω\OmegaΩ was defined as τ{pU∈℘U∣pU is inductive}\tau \left \{ pU \in \wp U | pU \, is \, inductive \right \}τ{pU∈℘U∣pUisinductive}, this simplifies to having to show that if {w∣τ(σw)∈pU}\left \{w | \tau (\sigma w) \in pU \right \}{w∣τ(σw)∈pU} is inductive, then Ω\OmegaΩ is in {w∣τ(σw)∈pU}\left \{w | \tau (\sigma w) \in pU \right \}{w∣τ(σw)∈pU}. But that follows directly from our initial assumption that Ω\OmegaΩ is well founded. Therefore our proof is complete.

A term of the empty type

With both a proof that Ω\OmegaΩ is well founded and a proof that Ω\OmegaΩ is not well founded in hand, we can at last construct the term of type ⊥\bot⊥:

false : ⊥
false = ¬well-founded-Ω well-founded-Ω

This concludes the proof that λΠττ\lambda_\Pi^{\tau \tau}λΠττ​ is inconsistent.

A Hierarchy of Sorts

As evident from the construction of a contradiction in λΠττ\lambda_\Pi^{\tau \tau}λΠττ​ presented above, it is necessary for the expressiveness of our dependently typed lambda calculus to be weakened in some way for it to be consistent. However, we simultaneously do not want to give up the ability to express propositions of practical interest and their proofs in our lambda calculus.

Luckily, a rather simple modification to the type system of λΠττ\lambda_\Pi^{\tau \tau}λΠττ​ is sufficient to make it consistent again [Martin-Löf 1973][Coquand 1986], replacing the problematic type rule ∗:∗\ast : \ast∗:∗ by a hierarchy of sorts:

∗:∗1∗1:∗2∗2:∗3∗3:∗4...\begin{aligned} \ast & : \ast_1\\ \ast_1 & : \ast_2\\ \ast_2 & : \ast_3\\ \ast_3 & : \ast_4\\ & ... \end{aligned} ∗∗1​∗2​∗3​​:∗1​:∗2​:∗3​:∗4​...​

So just like in λΠττ\lambda_\Pi^{\tau \tau}λΠττ​ every object term, like truetruetrue, has some type, like BoolBoolBool, and every type term has the kind ∗\ast∗. However, the term ∗\ast∗ itself does not have the kind ∗\ast∗, but the sort ∗1\ast_1∗1​, the term ∗1\ast_1∗1​ has the sort ∗2\ast_2∗2​, the term ∗2\ast_2∗2​ has the sort ∗3\ast_3∗3​ and so on.

In the following, we will use for consistency instead of ∗\ast∗ as the sort of types ∗0\ast_0∗0​.

This lambda calculus we shall call λΠω\lambda_\Pi^\omegaλΠω​. The grammar and type rules for it are as follows:

e,ρ::=e:ρ∣x∣e  e′∣λx.e∣∗ℓ∣∀x:ρ.ρ′\begin{aligned} e , \rho ::= & \quad e : \rho\\ | & \quad x\\ | & \quad e \; e'\\ | & \quad \lambda x . e\\ | & \quad \ast_\ell\\ | & \quad \forall x : \rho . \rho' \end{aligned} e,ρ::=∣∣∣∣∣​e:ρxee′λx.e∗ℓ​∀x:ρ.ρ′​

Γ⊢e:↑τΓ⊢e:↓τ\frac{\Gamma \vdash e :_\uparrow \tau} {\Gamma \vdash e :_\downarrow \tau} Γ⊢e:↓​τΓ⊢e:↑​τ​

Γ(x)=τΓ⊢x:↑τ\frac{\Gamma(x) = \tau} {\Gamma \vdash x :_\uparrow \tau} Γ⊢x:↑​τΓ(x)=τ​

Γ,x:τ⊢e:↓τ′Γ⊢λx.e:↓∀x:τ.τ′\frac{\Gamma , x : \tau \vdash e :_\downarrow \tau'} {\Gamma \vdash \lambda x . e :_\downarrow \forall x : \tau . \tau'} Γ⊢λx.e:↓​∀x:τ.τ′Γ,x:τ⊢e:↓​τ′​

Γ⊢e:↑∀x:τ.τ′Γ⊢e′:↓τΓ⊢e  e′:↑τ′ ⁣[ x↦e′ ]\frac{\Gamma \vdash e :_\uparrow \forall x : \tau . \tau' \quad \Gamma \vdash e' :_\downarrow \tau} {\Gamma \vdash e \; e' :_\uparrow \tau' \! \left [ \, x \mapsto e' \, \right ]} Γ⊢ee′:↑​τ′[x↦e′]Γ⊢e:↑​∀x:τ.τ′Γ⊢e′:↓​τ​

Only the last three rules differ from λΠττ\lambda_\Pi^{\tau \tau}λΠττ​, and only the last two do so substantially. Also of note is that some type judgements have turned from type checking to type inference due to the need to know the level ℓ\ellℓ for a sort ∗ℓ\ast_\ell∗ℓ​, which will in turn necessicate corresponding adaptations in the implementation.

The last type rule, ∗ℓ:∗ℓ+1\ast_\ell : \ast_{\ell + 1}∗ℓ​:∗ℓ+1​, is a direct implementation of the hierarchy of sorts as explained above.

However of course it is also necessary to reconsider type judgements over terms ∀x:ρ.ρ′\forall x : \rho . \rho'∀x:ρ.ρ′ in the second to last type rule.

One option in defining the type rule for ∀x:ρ.ρ′\forall x : \rho . \rho'∀x:ρ.ρ′ would be to simply directly keep the rule from λΠττ\lambda_\Pi^{\tau \tau}λΠττ​ in our new type system:

Γ⊢ρ:↓∗0ρ⇓τΓ,x:τ⊢ρ′:↓∗0Γ⊢∀x:ρ.ρ′:↑∗0\frac{\Gamma \vdash \rho :_\downarrow \ast_0 \quad \rho \Downarrow \tau \quad \Gamma , x : \tau \vdash \rho' :_\downarrow \ast_0} {\Gamma \vdash \forall x : \rho . \rho' :_\uparrow \ast_0} Γ⊢∀x:ρ.ρ′:↑​∗0​Γ⊢ρ:↓​∗0​ρ⇓τΓ,x:τ⊢ρ′:↓​∗0​​

While this would be consistent (since it is simply a strictly less powerful type system than that of λΠω\lambda_\Pi^\omegaλΠω​), and would have the advantage of that we would only have to check that the kind of ρ\rhoρ and ρ′\rho'ρ′ is ∗0\ast_0∗0​, it would greatly restrict the expressiveness of our language, since it would mean that a function could not take a type as an argument or return a type as it's result, it could only go from objects to objects.

Instead we will allow for functions to go from any sort to any sort, from objects to types, from types to objects, from kinds to objects, from objects to kinds, etc. :

Γ⊢ρ:↑∗ℓρ⇓τΓ,x:τ⊢ρ′:↑∗ℓ′Γ⊢∀x:ρ.ρ′:↑∗max(ℓ,ℓ′)\frac{\Gamma \vdash \rho :_\uparrow \ast_\ell \quad \rho \Downarrow \tau \quad \Gamma , x : \tau \vdash \rho' :_\uparrow \ast_{\ell'}} {\Gamma \vdash \forall x : \rho . \rho' :_\uparrow \ast_{max(\ell, \ell')}} Γ⊢∀x:ρ.ρ′:↑​∗max(ℓ,ℓ′)​Γ⊢ρ:↑​∗ℓ​ρ⇓τΓ,x:τ⊢ρ′:↑​∗ℓ′​​

So the sort of some term ∀x:ρ.ρ′\forall x : \rho . \rho'∀x:ρ.ρ′ is simply the higher of the sorts of ρ\rhoρ and ρ′\rho'ρ′.

Taking ρ→ρ′\rho \rightarrow \rho'ρ→ρ′ as a shorthand for ∀x:ρ.ρ′\forall x:\rho . \rho'∀x:ρ.ρ′ with xxx not apeparing in ρ′\rho'ρ′, this means for example that a term ∗7→∗3\ast_7 \rightarrow \ast_3∗7​→∗3​ is of the sort ∗8\ast_8∗8​, since ∗7:∗8\ast_7 : \ast_8∗7​:∗8​ and ∗3:∗4\ast_3 : \ast_4∗3​:∗4​.

As a more practical example, the "powerset" function ℘\wp℘ we defined above for our construction of Hurkens' paradox, ℘A:=A→∗0\wp A := A \rightarrow \ast_0℘A:=A→∗0​, would have the sort ∗0→∗1\ast_0 \rightarrow \ast_1∗0​→∗1​.

This might already hint towards how the construction of Hurkens' paradox presented above will no longer work in λΠω\lambda_\Pi^\omegaλΠω​, given that it heavily relies on the fact that in λΠττ\lambda_{\Pi}^{\tau \tau}λΠττ​, ℘:∗→∗\wp : \ast \rightarrow \ast℘:∗→∗.

However, the definition of the function ℘\wp℘ above might also raise the question what to do if we do not want to look at the type of all propositions over some type, but rather over some kind, or over some higher sort. Do we define ℘1K:=K→∗0\wp_1 K := K \rightarrow \ast_0℘1​K:=K→∗0​ of sort ∗1→∗2\ast_1 \rightarrow \ast_2∗1​→∗2​, ℘2S:=S→∗0\wp_2 S := S \rightarrow \ast_0℘2​S:=S→∗0​ of sort ∗2→∗3\ast_2 \rightarrow \ast_3∗2​→∗3​, and so on?

In the type system presented here, we do not have any other choice but to go the above route. However, there are possible extensions to alleviate this redundancy of definitions, namely universe polymorphism [Sozeau et al. 2014]. However not only does this require levels to become terms inside the lambda calculus itself, but also brings with it the need to introduce a second hierarchy of sorts for level polymorphic function types. We will not implement this here.

Implementation

With the introduction of hierarchy of sorts having necessitated that some judgements in our type rules have turned from type checking to type inference, the implementation of λΠττ\lambda_\Pi^{\tau \tau}λΠττ​ has to be changed in quite a few places. However, most of these changes are not terribly significant. Rather, we will focus here on the changes to the implementation due to the new type rules for  e:ρ \,e : \rho\,e:ρ, ∀x:ρ.ρ′\forall x : \rho . \rho'∀x:ρ.ρ′, and ∗ℓ\ast_\ell∗ℓ​.

The Abstract Syntax Tree Data Type

data TermInfer
  = Ann TermCheck TermInfer
  | Star Int
  | Pi TermInfer TermInfer
  | Bound Int
  | Free Name
  | TermInfer :@: TermCheck
  deriving (Show, Eq)

Star now no longer simply is a constant constructor, but has an argument, it's level. And both Ann and Pi now have in places a TermCheck replaced with TermInfer due to the necessary changes to the type rules.

typeInfer for Ann

typeInfer i g (Ann e r) =
  do
    s <- typeInfer i g r
    case s of
      (VStar l) -> do
        let t = evalInfer [] r
        typeCheck i g e t
        return t
      _ -> failure ":("

We infer the type of r, which has to be some VStar l or we fail. Otherwise we proceed as in the implementation of λΠττ\lambda_\Pi^{\tau \tau}λΠττ​, evaluating r to t and checking e against t.

typeInfer for Pi

typeInfer i g (Pi r r') =
  do
    s <- typeInfer i g r
    case s of
      (VStar l) -> do
        let t = evalInfer [] r
        s' <-
          typeInfer
            (i + 1)
            ((Local i, t) : g)
            (substInfer 0 (Free (Local i)) r')
        case s' of
          (VStar l') -> return (VStar (max l l'))
          _ -> failure ":("
      _ -> failure ":("

We infer the type for r, which has to be some VStar l or we fail. Otherwise, we evaluate r to t and infer the type of r' in the extended context, which again has to be some VStar l'. With both the sort level l of r and l' of r' determined, we can return the sort of our Pi term, VStar (max l l').

And that is all.
